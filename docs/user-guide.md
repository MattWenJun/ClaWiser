# ClaWiser 用户使用手册：让你的 AI 助手拥有“长期记忆”与“深度逻辑”

## 1. 写在前面：ClaWiser 能为你做什么？

欢迎使用 **ClaWiser**。作为一名技术产品经理，我深知在复杂办公场景中，AI 助手往往受限于“瞬时记忆”和“线性思维”，难以处理跨天迭代的深度任务。

ClaWiser 的核心价值在于：通过 8 个互补的模块，将你的 AI 从一个简单的“聊天机器人”进化为拥有职业素养的“资深项目伙伴”。它不仅能记住你的决策偏好，更能像专业人士一样管理工作流。这些能力被分为两层，安装即用：

| 类别 | 包含模块 | 核心价值 |
| :--- | :--- | :--- |
| **工作流层 (日常互动)** | `hdd`, `product-first`, `save-game`, `load-game`, `project-skill-pairing` | 规范 AI 思考逻辑，确保项目进度可见、技能归属清晰。 |
| **记忆层 (底层支持)** | `memory-deposit`, `retrieval-enhance`, `noise-reduction` | 打造 6 层记忆架构，提升搜索准确度并自动清洗冗余数据。 |

## 2. HDD：假设驱动开发——拒绝“盲目尝试”，先验证再动手

在处理 Debug 或技术方案选型时，AI 常因急于写代码而陷入“试错循环”。HDD (Hypothesis-Driven Development) 模块强制 AI 采用科学实验逻辑：

*   **核心功能：** 通过“提出假设-验证假设”的闭环，解决高不确定性任务。
*   **触发场景：** Bug 诊断、方案优选、复杂重构、架构变更。
*   **激活暗号：** > [!IMPORTANT]
    > **“用 HDD”**、**“先验证假设”**、**“按 TDD 方式”**

### 核心操作流程（0-6 七步法）
*   **Step 0 资源映射 (关键！)：** 动手前必须清点战场。AI 会先问：“我有哪些文档、API key 或配置可用？”避免在错误的解空间里盲目摸索。
*   **HT (假设验证)：** 提出假设并进行**自我攻击**（检查是否有反例、是否具备其他解释）。
*   **退避路径 (Retreat Rule)：** 若假设失败，**必须替换假设**，而非修补错误方案。
*   **PT (方案验证)：** 验证技术实现的可行性。
*   **AT Design & Run：** 先写验收测试，再跑代码。
*   **E2E：** 端到端交付。

**TPM 进阶 Tips：** 善用 HDD 的“异常模式识别”。如果 AI 发现计数不对或不自然的一致性（如分类器全部分为一类），它会优先分析模式而非单条数据。简单的文案修改无需使用此模块。

## 3. Product-First (PF)：产品先行——技术实现前，先定义用户价值

很多项目失败是因为“技术先行”导致功能堆砌。PF 模块确保我们在写代码前，先达成产品层面的共识（Alignment）。

*   **激活暗号：** > [!IMPORTANT]
    > **“做个新项目”**、**“做产品规划”**、**“定义用户场景”**、**“PF”**
*   **核心产出：** 会在 `memory/projects/<项目名>/` 自动生成 **`product-plan.md`**。
*   **产品计划三要素：**
    1.  **用户场景：** 必须描述“用户一天的使用故事”，而非功能列表。
    2.  **判断依据：** 定义什么才算“好”的质量标准。
    3.  **Phase 划分：** 按用户价值（MVP 最小可用集）拆分，而非按技术模块拆分。

### 避坑指南
| 常见陷阱 | Senior TPM 的正确做法 |
| :--- | :--- |
| **技术驱动：** 先想数据库怎么存，再想界面。 | **场景驱动：** 先定义用户看到什么，反推数据需求。 |
| **功能堆砌：** Phase 1 就想做完所有 feature。 | **MVP 逻辑：** Phase 1 只保留跑通核心场景的最小集。 |

## 4. Save-Game：项目存档——随时停下的“进度保险”

存档不只是记录状态，它是一轮深度的“产出复盘”与“上下文管理”。

*   **触发场景：** 工作段结束、Context（上下文）快满、任务移交。
*   **激活暗号：** > [!IMPORTANT]
    > **“存档”**、**“记一下进度”**、**“Save Game”**、**“写交接”**
*   **存档核心逻辑：**
    1.  **复盘偏差：** 对比 `product-plan.md`，识别计划外工作。
    2.  **提炼“铁律”：** 记录踩过的坑，防止未来的 AI 重蹈覆辙。
    3.  **Step 3.5 产出封装检查：** TPM 建议——不要在 context 快满时立即封装 Skill，应在 `HANDOFF.md` 中记录为“待办”，避免对话因过载而崩溃。
*   **产出物：** 结构化的 `HANDOFF.md`，包含目标、进度、下一步和关键数字（ID/端口等）。

---

## 5. Load-Game：项目读档——无缝衔接，瞬间找回状态

当对话发生 **Compaction (记忆压缩)** 或跨天恢复时，读档是恢复“高保真”状态的唯一途径。

*   **激活暗号：** > [!IMPORTANT]
    > **“读档”**、**“恢复项目”**、**“review 进度”**
*   **TPM 逻辑：** 读档不只是“读”，而是“验证”。AI 会检查 `HANDOFF.md` 中的 P0（阻塞项）、P1（质量项）、P2（优化项），并在开工前通过命令核实实际状态。
*   **注意：** 发生 Compaction 后，`HANDOFF.md` 的优先级高于系统的摘要（Summary）。

## 6. Project-Skill-Pairing：项目与 Skill 结对——有序的工程管理

避免技能碎片化，确保每一个自定义 Skill 都有其关联的项目背景。

*   **项目分级 (Tiers)：**
    *   **Tier 1 (轻量)：** 一次性修复或探索任务。
    *   **Tier 2 (标准)：** 跨天迭代，有明确 Phase 划分。
    *   **Tier 3 (复杂)：** 长期维护 (>1个月)，包含多子系统，需维护 `README.md` 和架构图。
*   **管理规则：** 新建项目衍生 Skill 时，AI 会自动在项目文件夹下建立 **Symlink (符号链接)**，实现双向绑定。

## 7. Memory-Deposit：记忆沉淀——你的 AI “外挂大脑”

这是 ClaWiser 的核心地基，通过 6 层记忆系统解决“转头就忘”的问题。

*   **核心机制：** 
    1.  **对话合并：** 通过 `merge-daily-transcript.js` 自动将对话整理为转录存档。
    2.  **Git 版本管理：** 利用 `auto-commit.sh` 追踪文件变动，决策逻辑可追溯。
    3.  **向量召回：** 当 AI 提示你需要 **“Embedding API key”** 时请积极配合，它是实现秒级精准搜索的前提。
*   **文件分工：**
    *   `MEMORY.md`：记录稳定的偏好、习惯和长期模式。
    *   `每日笔记 (memory/YYYY-MM-DD.md)`：记录具体的决策理由和当日教训。

## 8. Noise-Reduction：数据降噪——洗掉废话，提高信噪比

AI 记忆力下降往往是因为记住了太多“垃圾信息”。

*   **核心定义：** **信号**是用户与 Agent 的真实对话；**噪声**是管道数据（JSON 输出、Heartbeat 心跳、工具调用回显）。
*   **运作流程：** 诊断噪声画像 -> 编写降噪规则 -> 验证压缩率。
*   **质量指标：** 健康的**压缩率通常在 30%-60%**。如果搜不到以前的内容，通常说明需要运行“降噪”了。
*   **激活暗号：** **“降噪”**、**“清洗对话”**、**“提高信噪比”**。

---

## 9. Retrieval-Enhance：搜索增强——静默运行的资深分析师

这是一个默默守护的后台模块，无需用户手动触发。

*   **查询扩展 (Query Expansion)：** 当你提出模糊需求时，它会生成 4 个变体（同义词、中英切换、角度变换）进行**并行搜索**。
*   **Self-Reranking：** 它会从搜索到的 Top 5-10 结果中，基于你的真实意图进行二次重排。
*   **用户感知：** 当你感叹“我半个月前随口提的需求你居然都记得”时，就是它在发挥作用。

## 10. 常见问题与进阶 Tips

*   **模块组合拳：**
    1.  **启动：** 用 **PF** 定义目标，生成 `product-plan.md`。
    2.  **攻坚：** 遇到技术难题，用 **HDD** 的 Step 0 开始资源映射。
    3.  **收尾：** 下班前用 **Save-Game** 存档。
    4.  **复工：** 第二天用 **Load-Game** 检查 P0/P1 偏差。
*   **随时问诊：** 你可以随时用自然语言询问：“你现在能用哪些 ClaWiser 模块？”或“帮我检查一下当前项目的 Tier 级别是否需要上调”。
